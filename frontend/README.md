# Frontend Engineer Challenge

Frontend проект для 3 сценариев:

- Регистрация
- Авторизация
- Восстановление пароля

## Быстрый старт

```bash
# Установка зависимостей
npm install
# Запуск для разработки
npm run dev
```

## Документация компонентов

```bash
npm run storybook
```

## Навигация по проекту

```text
src/
├── app/         — точка входа, здесь происходит запуск проекта, глобальная конфигурация, связь фич в единое приложение
├── features/    — крупные куски функциональности проекта, включая страницы
├── services/    — часть функционала, которая используется в нескольких features
├── shared/      — утилиты, глобальный UI-kit, общие конфиги
```

### Алиасы

Зарегистрированы в `vite.config.ts` и `tsconfig`:

- `@` → `src/`

### Что можно, а что не стоит делать

|                 | 🟢 Можно                                                                             | 🔴 Лучше не стоит                   |
| --------------- | ------------------------------------------------------------------------------------ | ----------------------------------- |
| `app`           | добавлять провайдеры и регистрировать страницы                                       | создавть компоненты                 |
| `features/*`    | добавлять страницы и фичи                                                            | делать перекрессные ссылки          |
| `services/*`    | виджеты и фитчи, которые используются в нескольких фитчах, делать перекресные ссылки |                                     |
| `shared/assets` | добавлять общие иконки, картинки, шрифты шрифты                                      | добавлять код                       |
| `shared/lib`    | добавлять общие утилиты                                                              | добалять компоненты                 |
| `shared/model`  | добавлять общие типы, конфиги хуки и стор                                            | добалять компоненты                 |
| `shared/ui`     | добавлять ui-kit компоненты                                                          | добавлять что-то помимо компонентов |

## О чём помнить при разработке

- Один файл — один именованный экспорт
- Aliases обязательны для всех файлов, расположенных в features и services `../../../../`
- index.ts — только для реэкспорта
- Все имена файлов имееют вид `some-file.ts`
- Все хуки именуются как `useSomething`, на имена файлов это правило не распространяется
- Все сторы именуются как `useSomethingStore`
- Все компоненты именуются как `SomePopupComponent`
- Все стили — через `app/index.css`, Tailwind
- Строгий ESLint. Все автофиксы можно прогнать через `npm run lint`
- SVG иконки кладём в папку `shared/assets/icons/...`
- Используй `cn` для динамических вариантов
- public api желателен для всех файлов, расположенных в features и services

## Формат коммитов

Формат:

```text
type(feature): description
```

- _type_ - представляет собой символ
  - "+" - новая задача
  - "=" - правка
  - "~" - минорное изменение, на которое можно не обращать внимания
- _feature_ - название функционала, который бил добавлен или изменен по сути, это имя папки в фитчах или shared
- _description_ - описание изменений

Примеры:

```text
+auth: добавлена страница логина
```

```text
=auth: ошибки валидации не отображались
```

```text
~ui-kit: поправлены проблемы стиля кнопки
```
